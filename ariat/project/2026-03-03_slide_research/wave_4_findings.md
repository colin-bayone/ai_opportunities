### Team Structure Model

**Phased Ramp:**

| Period | Onshore | Offshore | Total | Focus |
|--------|---------|----------|-------|-------|
| Q1 | 1 Senior | 2 Engineers | 3-4 | Discovery + Foundation |
| Q2 | 1 Senior | 4 Engineers | 5 | Unified Interface |

| Quarter | Cost | Budget | Buffer |

| Q1 | $71.5K | $100K | $28.5K |

**Key Principle:** "Successful delivery over cost/margin. Size the team for success first."

### Patterns for Ariat GCC

2. **Onshore/offshore hybrid:** 1 onshore (client presence) + 4-5 offshore (implementation)
3. **Specialization by domain:** Expert in their domain, not junior levels
4. **Critical path hiring:** Identify rare skills early (Agentic AI = 2-3 months lead time)
5. **Discovery investment:** 40-80 hours pre-engagement discovery

## Folder 3: ui-conversion-discovery

### What It Contains
- UI conversion POC engagement (EPNM → EMS)
- 5 proposal iterations with quality critique
- Technical research on framework translation
- AI-accelerated modernization approach

### Business Context

- **Problem:** 70-100 legacy UI screens need conversion
- **Challenge:** Customers refuse to migrate due to UI familiarity
- **POC Scope:** 2-3 screens in 4 weeks
- **Architecture:** Monolith (Dojo/Java) → Microservices (Angular/Java)

### AI-Accelerated Approach

**Phase 1: Exploration (AI-Driven)**
- Claude Code for codebase analysis
- Automated mapping: module boundaries, dependencies, patterns
- Screen categorization by complexity

**Phase 2: Conversion (AI-Assisted)**
- Custom skills and sub-agents
- Analysis → Planning → Execution → Validation
- Pattern library for consistency

**Scaled Engagement (LangGraph Multi-Agent):**
- Architect agent: strategy decisions
- Engineer agent: planning and design
- Foreman agent: spawning workers
- Judge agent: quality control

### Pattern Library Concept

- Conversion patterns (Dojo → Angular)
- Unit tests for edge cases
- Gap inventory (what exists vs. doesn't)
- Backend status tracking
- Dependency mapping
- Historical tracking for traceability

### "Flywheel" Acceleration

| Category | POC (Front-loaded) | Per-Screen (Repeatable) |
|----------|-------------------|------------------------|
| Pattern discovery | High investment | Minimal (patterns cataloged) |
| Agent tuning | Custom development | Incremental only |
| Implementation | First instances | Apply known transformations |
| Validation | Build test infrastructure | Execute existing comparisons |

### Relevance for Ariat

1. **Legacy modernization:** Same pattern for any old → new conversion
2. **Pattern-based acceleration:** Reusable across screens/modules
3. **AI-assisted development:** Demonstrates BayOne's technical depth
4. **Visual testing:** Playwright for automated validation
5. **POC scoping:** 2-3 items to prove capability, extrapolate rest
