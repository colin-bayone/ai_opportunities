# Role: Backend / Integration Engineer (Offshore)

**Location:** Offshore (India)

**Reports to:** Senior Engineer (Onshore), with oversight from Colin Moore (Director of AI)

**Relationship to Team:** Core implementation role and workhorse of the team. Works under the technical guidance of the onshore senior engineer but expected to be a strong partner, not just someone who takes orders. Responsible for building the data pipelines, integrations, and backend services that form the foundation of this engagement.

---

## Why This Role Exists

The majority of the work in this engagement involves connecting to Cisco's existing systems (Airflow, Jenkins, CAT, DevX, Grafana), building data pipelines, and creating backend services. This is the person who builds that infrastructure.

Items A (Developer Box), C (Unified Interface), and F (Branch Health) all require substantial backend and integration work. This role is active from Day 1 and remains critical throughout the engagement.

---

## Day-to-Day Responsibilities

**Data Pipelines:**
- Build telemetry systems to capture test runs, pass/fail status, duration, coverage (Item A)
- Create data pipelines to store, process, and surface results
- Work with Cisco's existing data sources and schemas

**API Integrations:**
- Integrate with CAT, DevX, Jenkins, Airflow, Grafana
- Normalize data across different systems and schemas
- Build the unified data layer that AI features depend on

**Backend Services:**
- Develop backend services using Python
- Create APIs for front-end and other services to consume
- Ensure services can run in an on-prem environment

**Collaboration:**
- Daily syncs with onshore senior engineer
- Coordinate with front-end engineer on API contracts
- Coordinate with AI/ML engineer on data requirements for AI features
- Proactive communication about blockers and progress

---

- Strong Python experience
- **Airflow experience required** (critical - this is the main orchestration tool)

- Familiarity with CI/CD workflows and automation

**AI and Tooling:**
- Proficient with AI pair programming tools beyond basic Copilot (Claude Code, Cursor, or similar)
- Expected to use AI tools daily to accelerate development
- Familiar with AI/ML concepts (not building AI features, but should understand them)
- Comfortable working alongside AI engineers and supporting their data needs

**Working Style:**
- Can work independently with guidance from onshore lead
- Proactive communicator, especially across time zones
- Comfortable with ambiguity; can figure things out
- Detail-oriented; integrations require precision
