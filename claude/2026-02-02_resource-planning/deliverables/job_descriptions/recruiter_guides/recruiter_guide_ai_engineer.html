Recruiter Guide

AI Engineer

Remote (Offshore) | Chat Interface + Dashboard Focus
Helpful for pre-screening: If your screening call is on Teams, please enable transcription and share the transcript afterward. This helps our technical team review candidates more effectively. Not required, but appreciated!
Must-Have Skills
  • 4-6+ years AI/NLP production experience
    Built AI systems that real users relied on, not just research or coursework
  • LangChain or LangGraph
    Orchestration frameworks for building AI pipelines; understands chains and agents
  • Python backend development
    FastAPI or Django, APIs, data processing pipelines
  • Prompt engineering
    Context management, few-shot learning, structured outputs, handling when AI gives bad answers
  • Chat interface experience
    Built conversational AI or natural language query systems; conversation history, multi-turn context
  • RAG and vector search
    Built retrieval systems with pgvector, Pinecone, or similar; understands embeddings
  • On-prem deployment capability
    Can deploy and maintain AI services without relying on cloud-managed services
  • AI pair programming proficiency
    Actively uses Claude Code, Cursor, or similar agentic AI tools daily. Not just autocomplete (Copilot) - should be comfortable with AI that writes and edits code autonomously
Nice-to-Have Background
  • Dashboard / data visualization
    Built monitoring dashboards or analytics interfaces (React, Streamlit, Grafana)
  • Log analysis experience
    Parsed logs, analyzed failures, built tools that help debug issues
  • CI/CD familiarity
    Understands build pipelines, test automation, deployment workflows
  • Streaming/real-time
    WebSockets, server-sent events, streaming LLM responses
  • NLP fundamentals
    Text processing, parsing, search beyond just LLM APIs
  • Strong async communication
    Remote work requires clear written updates across time zones
Strong Signals
  • Built chat interfaces or conversational AI that shipped to production
  • Enthusiastic about AI tools - can describe specific ways Claude Code or Cursor changed their workflow
  • Dashboard or data visualization experience
  • Strong written communication (critical for remote/offshore)
  • Experience working with distributed teams across time zones
  • Has deployed AI services themselves, not just handed off to DevOps
Warning Signs
  • Only notebook/research experience - never deployed for real users
  • Can't explain how they test AI systems with variable outputs
  • Poor written communication (critical for remote work)
  • Doesn't use AI coding tools, or only uses basic autocomplete (Copilot alone)
  • Needs constant direction, not self-starting
  • Vague about what they personally built vs what the team built
Screening Questions

Record all screening calls. These questions reveal depth without requiring technical evaluation. Note answers verbatim for technical review.

"Describe an AI-powered feature you built that went to production. What did it do and who used it?"
Weak Answer
Vague description without specifics. Describes a demo, prototype, or hackathon project. Says "the team built it" without clarifying their personal role.
Good Answer
Specific feature with concrete details - what problem it solved, what technologies they used, how many users or what scale. Clear about what THEY built vs the team.
"How do you test AI features when the outputs aren't deterministic?"
Weak Answer
Can't explain an approach, or says "you can't really test AI." No structured methodology.
Good Answer
Describes specific approaches - evaluation datasets, human review processes, regression testing on known examples, monitoring production quality. Has actually thought about this problem.
"Have you built any chat interfaces? Walk me through how conversation history and context work."
Weak Answer
Never built chat interfaces, or can't explain how they handle multi-turn conversations. Vague about architecture.
Good Answer
Explains how they store conversation history, manage context windows, handle when context gets too long. Can describe real implementation decisions they made.
"What AI coding tools do you use, and how have they changed the way you work?"
Weak Answer
Doesn't use AI tools, or only mentions basic autocomplete like Copilot. Can't describe specific ways it changed their workflow. Dismissive about AI assistance.
Good Answer
Uses Claude Code, Cursor, or similar agentic tools. Gives specific examples of how it changed their work - faster debugging, writing tests, exploring unfamiliar codebases. Enthusiastic about the productivity gains.
"How do you stay on track and communicate progress when working remotely with a team in different time zones?"
Weak Answer
No experience with remote/distributed work. Depends on synchronous meetings for everything. Poor at written updates.
Good Answer
Describes specific practices - written daily updates, async documentation, proactive communication about blockers. Has worked this way before and has systems that work.
"Can you explain [most complex thing on their resume] to me like I'm not technical?"
Weak Answer
Uses jargon, gets frustrated, or can't simplify it.
Good Answer
Uses plain language and analogies. Checks if you're following. Patient and clear. (You can evaluate this directly.)

    Not just research, demos, or internal experiments - actual products users relied on
