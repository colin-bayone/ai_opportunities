Basic Subgraph with Title

Subgraphs create visual groupings (clusters) around related nodes. Syntax: subgraph ID["Label"]...end. The ID is used for styling, the label is displayed.

graph TB subgraph FRONTEND["Frontend Layer"] A["React App"] B["API Client"] end subgraph BACKEND["Backend Layer"] C["Django Views"] D["Services"] end subgraph DATA["Data Layer"] E[("PostgreSQL")] F[("Redis Cache")] end A --> B --> C --> D D --> E D --> F style FRONTEND fill:#eff6ff,stroke:#2563eb,stroke-width:2px,color:#1e3a5f style BACKEND fill:#dbeafe,stroke:#3b82f6,stroke-width:2px,color:#1e3a5f style DATA fill:#fef9c3,stroke:#ca8a04,stroke-width:2px,color:#713f12

Nested Subgraphs (2 Levels)

Subgraphs can be nested to create hierarchical groupings. Keep nesting to 2-3 levels maximum for readability.

graph TB subgraph CLOUD["Azure Cloud"] subgraph AKS["AKS Cluster"] WEB["Web Pod"] WORKER["Worker Pod"] end subgraph MANAGED["Managed Services"] DB[("SQL Database")] BLOB[("Blob Storage")] end end WEB --> WORKER WORKER --> DB WORKER --> BLOB style CLOUD fill:#f0f9ff,stroke:#0284c7,stroke-width:3px,color:#0c4a6e style AKS fill:#dbeafe,stroke:#3b82f6,stroke-width:2px,color:#1e3a5f style MANAGED fill:#fef9c3,stroke:#ca8a04,stroke-width:2px,color:#713f12

Direction Override Within Subgraphs

The parent graph flows top-to-bottom (TB), but each subgraph can override its internal direction with direction LR. This creates a mixed layout.

graph TB subgraph INPUT["Input Sources"] direction LR EMAIL["Email"] API["API"] UPLOAD["Upload"] end PROC{{"Processor"}} subgraph OUTPUT["Output Targets"] direction LR DB[("Database")] QUEUE["Queue"] LOG["Log"] end INPUT --> PROC --> OUTPUT style INPUT fill:#dcfce7,stroke:#16a34a,stroke-width:2px,color:#14532d style OUTPUT fill:#ede9fe,stroke:#7c3aed,stroke-width:2px,color:#3b0764

Subgraph Syntax Reference

subgraph ID["Label Text"] direction TB %% or LR — overrides parent graph direction A["Node inside"] B["Another node"] A --> B end %% Style the subgraph cluster style ID fill:#eff6ff,stroke:#2563eb,stroke-width:2px,color:#1e3a5f %% Nested subgraphs subgraph OUTER["Outer"] subgraph INNER["Inner"] C["Deeply nested"] end end %% Labels support HTML (with htmlLabels: true) subgraph FANCY["<b>BOLD TITLE</b>"] D["Node"] end

Post-Render Cluster Label Fix

SVG renders elements in DOM order. Cluster (subgraph) labels render before edges, so arrows can cut through titles. The fix: after mermaid renders, clone label elements to the end of the SVG so they paint on top of edges.

When to use: Only needed when edges visually cross through subgraph title text. If your diagram doesn't have this issue, skip the fix.

Cluster Label Fix — JavaScript

// Run AFTER mermaid.run() completes const svg = document.querySelector('.mermaid svg'); if (svg) { document.querySelectorAll('.cluster-label').forEach(label => { const fo = label.querySelector('foreignObject'); if (fo) { const g = document.createElementNS('http://www.w3.org/2000/svg', 'g'); // Get position from the foreignObject const x = parseFloat(fo.getAttribute('x')) || 0; const y = parseFloat(fo.getAttribute('y')) || 0; const w = parseFloat(fo.getAttribute('width')) || 0; const h = parseFloat(fo.getAttribute('height')) || 0; // Find the parent cluster's stroke color const cluster = label.closest('.cluster'); let strokeColor = '#94a3b8'; if (cluster) { const clusterRect = cluster.querySelector('rect'); if (clusterRect) { strokeColor = clusterRect.getAttribute('stroke') || strokeColor; } } // White background rect with matching border const bg = document.createElementNS('http://www.w3.org/2000/svg', 'rect'); bg.setAttribute('x', x - 6); bg.setAttribute('y', y - 2); bg.setAttribute('width', w + 12); bg.setAttribute('height', h + 4); bg.setAttribute('rx', '4'); bg.setAttribute('fill', 'white'); bg.setAttribute('stroke', strokeColor); bg.setAttribute('stroke-width', '1'); g.appendChild(bg); // Clone the foreignObject (the label text) g.appendChild(fo.cloneNode(true)); // Copy transform from original label group const transform = label.getAttribute('transform'); if (transform) g.setAttribute('transform', transform); // Append to end of SVG (renders on top) svg.appendChild(g); // Hide original to prevent double-rendering label.style.display = 'none'; } }); }
