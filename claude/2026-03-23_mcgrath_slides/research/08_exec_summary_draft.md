## Layout Recommendations

Based on the available BayOne page layouts (from the Ariat deck system), these are the best fits for an executive summary slide:

### Top Pick: Problem to Solution Layout
- **Why it fits:** Natural narrative arc -- top section frames McGrath's challenge (the hook), bottom section presents BayOne's approach (the value prop + proof points)
- **Visual:** Muted gray challenge section on top, vibrant gradient solution section below
- **Content mapping:** 3 challenge bullets (complexity, simultaneity, SLA pressure) in the problem zone; 4-5 solution cards (AI ops, Oracle depth, discovery-first, integration expertise, SLA track record) in the solution zone

### Strong Alternative: Proof Point Layout
- **Why it fits:** "In Practice" badge establishes credibility immediately; 4 checkmark bullets map perfectly to proof points; stats footer anchors with numbers
- **Visual:** Left panel with solution categories, right panel with evidence and metrics
- **Content mapping:** Left panel lists BayOne's capabilities; right panel shows 4 proof bullets + bottom stats row (13 years, 100+ customers, 95.7% SLA)
- **Best paired with:** Version C (numbers-first framing)

### Backup: Split Screen Layout
- **Why it fits:** Left gradient panel carries the hook + 2 key stats; right panel holds 5 capability cards
- **Visual:** Clean 50/50 balance, strong left-side statement
- **Content mapping:** Left = opening hook + "13 years" + "95.7% SLA"; Right = 5 cards covering Oracle, AI, integrations, discovery, partnership

### Avoid for This Slide
- **Timeline Journey:** Implies phasing/process -- wrong for a summary
- **Two-Column Detail Rows:** Too granular for an opening slide
- **Dark Mode:** Equal-weight cards don't create the narrative hierarchy an exec summary needs
