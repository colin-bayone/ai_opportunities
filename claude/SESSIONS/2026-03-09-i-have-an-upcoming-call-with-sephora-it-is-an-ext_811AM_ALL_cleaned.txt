  Cognos Integration: Multiple Validated Paths

  Good news - there are several well-documented ways to access Cognos 10.2 programmatically:

  Path 1: IBM Cognos SDK (Java/.NET)

  The https://www.ironsidegroup.com/blog/sdk-bridging-the-gaps/. It can execute "literally all the functionality within the Cognos application" including:
  - Query the Content Store using XPath-like search paths
  - Access report definitions programmatically
  - Retrieve metadata about reports, folders, data sources

  Version note: For 10.2.x, the Java libraries must match the server version, but this is well-documented.

  Path 2: Content Store Database (Direct SQL)

  The Content Store is just a database (SQL Server or Oracle) containing all report definitions, folder hierarchy, and metadata. https://cogknowhow.com/t/cognos-content-store-database-tables-description/906.

  Caveats:
  - IBM warns schema can change between versions
  - Not recommended for production applications
  - But for a one-time migration project, this is perfectly fine

  IBM also provides CMTools.exe (ships with Cognos) for Content Store access.

  Path 3: Report Specifications as XML

  Reports are stored as https://www.ibm.com/support/pages/how-importexport-report-specification-ibm-cognos-analytics that can be exported. The XML contains the full report structure and embedded SQL.

  Path 4: Deployment Export

  Content Administration can export entire folders or the full Content Store as deployment packages.
