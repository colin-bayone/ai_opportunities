Pricing Breakdown

Phase Detail
Discovery and Codebase Analysis $50,000 (10%)

The foundation for the entire engagement. EPNM has over 15 years of accumulated complexity with limited design documentation. Understanding the codebase requires systematic AI-assisted exploration, not document review.

  • EPNM architecture mapping: module boundaries, service dependencies, and data flow tracing
  • Complete screen inventory with complexity classification by type (data entry, dashboards, reports, configuration, monitoring)
  • EMS integration point identification: which existing EMS services support which EPNM functionality, and where gaps exist
  • Conversion pattern identification: recurring UI structures and their backend correlates, documented for reuse
  • Collaborative screen prioritization with Cisco SMEs, focusing on functionality not yet present in EMS
  • Inventory of screens already ported to EMS vs. screens remaining in EPNM only, ensuring no duplication of existing work
  • Estimation model: conversion velocity data and per-screen effort calibration to support delivery timeline planning and customer commitments
  • Architecture review document with findings, risk assessment, and recommendations
AI Tooling and Agent Development $100,000 (20%)

The investment that enables 100% AI-generated code. This is proprietary tooling built specifically for the EPNM/EMS codebase, not a generic conversion tool. It is what makes the full conversion feasible at this price point.

  • Purpose-built conversion agents tuned to EPNM and EMS architecture patterns
  • Dojo-to-Angular semantic translation engine: widget structures, declarative markup, and data binding mapped to Angular's component model and dependency injection
  • Backend extraction patterns for monolith-to-microservices service boundary adaptation
  • Automated visual validation pipeline (Playwright-based): baseline capture from EPNM with side-by-side comparison against converted EMS screens
  • Persistent conversion pattern library that grows with each screen converted
  • Test coverage gap analysis: identification of where automated tests should exist but do not, with new test generation where needed
  • Codebase knowledge base: architecture, dependencies, edge cases, and institutional patterns documented and accessible to all agents

Screen Conversion and Integration $275,000 (55%)

The core delivery: 240 to 260 screens converted end-to-end so that the customer experience is seamless and indistinguishable from the original EPNM interface. This is UX preservation, not UX redesign. Each screen is a vertical slice through the full stack, not a UI reskin. Early conversions establish patterns that accelerate subsequent screens, with each completed screen making the next one faster.

  • Frontend UI conversion: Dojo/JavaScript widget structures translated to Angular components with semantic understanding of intent, not syntactic find-and-replace
  • Backend logic extraction: business rules, validation, data aggregation, and transformation logic identified in the EPNM monolith and adapted for EMS microservices boundaries
  • EMS service integration: converted screens connected to existing EMS endpoints, with gap documentation where services do not yet exist
  • Data flow preservation: each screen's complete data path (UI interaction through service layer to data access) verified for functional equivalence
  • Reactive state management: Dojo data stores and publish-subscribe patterns mapped to Angular RxJS observables and event emitters
  • Targeted backend adaptation where EMS services require modification to support converted screens (service adaptation, not new service development)
  • Explicit gap documentation for any EPNM functionality requiring new EMS services, with sufficient detail for Cisco's team to build if desired
  • Pattern capture: each completed conversion enriches the library, accelerating subsequent screens within the same category
Quality Engineering and Validation $75,000 (15%)

Continuous quality assurance integrated throughout the development cycle, not a separate phase at the end. 100% AI-generated code with human-in-the-loop oversight at every stage, ensuring quality keeps pace with delivery velocity.

  • Generative AI-driven QE embedded in every conversion cycle: each screen is validated as it is converted, not batched for later
  • Human-in-the-loop review: architectural oversight and quality verification at every stage of the agentic workflow, with human engineers reviewing all AI-generated output
  • Automated visual validation: side-by-side EPNM vs. EMS screen comparison confirming the converted experience is indistinguishable from the original
  • Automated regression testing: ensures changes to later screens do not break earlier conversions
  • Integration testing with EMS environment to verify end-to-end functionality in context
  • Domain-specific validation for element management: network inventory, topology, and automation lifecycle workflows verified for correctness against EPNM behavior, addressing the specialized nature of this domain
  • Continuous gap analysis throughout development, not just at completion, to catch missing functionality early
  • Final acceptance validation and stakeholder review support during the stabilization period
  • Defect resolution throughout the engagement, not deferred to a separate maintenance phase
Pricing Basis

This engagement is priced at $500,000 based on our current understanding of the EPNM and EMS architectures as discussed with your team. Scope is based on approximately 240 to 260 screens as identified through collaborative prioritization with the Cisco team. The final screen inventory will be confirmed during discovery, and BayOne will accommodate reasonable variance from this estimate within the engagement. Significant changes to the screen count or complexity beyond what is identified during discovery would be addressed through the change request process.

During the discovery phase, we will gain firsthand understanding of codebase structure and screen complexity. If that analysis reveals opportunities to streamline delivery, we will share those findings proactively. Any formal adjustments would follow the change request process outlined in our engagement pricing proposal.
