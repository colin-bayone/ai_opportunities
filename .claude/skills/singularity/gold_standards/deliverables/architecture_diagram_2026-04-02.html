Agent Pipeline Architecture

A LangGraph StateGraph with 15 nodes, 5 deterministic quality gates, and conditional retry routing. The pipeline proceeds through orchestration, parallel parsing, pattern mapping, code generation, configuration, validation, and human approval.

Design Principle

Deterministic first, LLM second. Every gate runs Python and sqlglot checks at zero LLM cost before proceeding. LLM-based quality checks only run after deterministic checks pass. This minimizes token usage while maintaining high validation standards.

Source grounding at every gate. Generated table names, column names, and source file references are checked against the original artifacts. If a name does not exist in the source data, the gate flags it as potentially hallucinated before it propagates downstream.

Azure Component Map

Azure
Azure AI Foundry
Claude Opus 4.6 and Sonnet 4.6 model inference. Enterprise-grade security (same as Outlook/Teams). Significantly lower cost than commercial API. Quota-based rate limiting with configurable concurrency.
Azure
Azure Container Apps
Hosts the FastAPI application server and PostgreSQL sidecar. Serverless scaling, HTTPS by default, time-bound credential access for secure demo environments.
Azure
Azure Blob Storage
Four containers: source artifacts (DataStage XML, stored procedures, DDLs), pipeline input, target examples (Scala apps, YAML templates, HQL), and pipeline output (generated configs).
Open Source
LangGraph
StateGraph orchestration framework. Manages the 15-node pipeline with conditional edges, parallel fan-out, retry routing, and human-in-the-loop interrupts via checkpointing.
Open Source
PostgreSQL
LangGraph checkpoint persistence and pipeline run history. Enables resume from any pipeline state, run comparison, and historical audit trail.
Open Source
sqlglot
AST-based SQL parsing for deterministic validation. Extracts tables, columns, joins, CTEs from both T-SQL (source) and Spark SQL (generated). Powers all gate checks at zero LLM cost.
External
LangSmith
Full execution tracing with waterfall visualization. Every API call, tool invocation, and node transition is recorded. Enables deep technical audit of any pipeline run.
Azure Phase 2
MCP Connector (Cognos)
Custom Model Context Protocol server wrapping the IBM Cognos Java SDK. Provides programmatic agent access to report metadata, embedded SQL, and RBAC. Requires Sephora Cognos environment access.

Model Allocation

Opus handles reasoning-intensive stages. Sonnet handles formulaic and parallelizable tasks. Deterministic gates use zero LLM tokens.

Pipeline Stage Model Rationale
Orchestrator Sonnet Fast artifact classification and pipeline identification
DataStage Parser Opus Complex XML analysis requiring deep context understanding
SQL Interpreter Sonnet Batch parallel processing of stored procedures and views
DDL Parser Sonnet Schema extraction with deterministic sqlglot pre-processing
Pattern Mapper Opus Critical mapping step requiring full upstream context
Spark SQL Planner Opus Plans all stages and builds dependency graph from Scala source
SQL Generator (independent) Sonnet Parallel generation of stages with no upstream dependencies
SQL Generator (dependent) Opus Generates stages that depend on previously generated views
Config Generator (3 files) Sonnet Template-based generation, 3 parallel calls
Schema Validator Sonnet Cross-artifact comparison and scoring
All 5 Deterministic Gates None Python + sqlglot only, zero LLM cost
05

Security and Data Handling

Key Security Properties

All model inference runs through Azure AI Foundry. No data is sent to commercial API endpoints. Azure AI Foundry provides the same enterprise security guarantees as Azure Active Directory, Outlook, and Teams.

Source data stays in Azure Blob Storage. Sephora's DataStage XML, stored procedures, and schema files are uploaded to a private storage account. Generated outputs are written back to the same storage account.

Time-bound access. Demo environments are provisioned with credentials that expire automatically. The server shuts down after a defined period. Access can be restricted to specific individuals.

Read-only exploration. The proposed exploration phase requires only read access to Sephora's systems. No writes, no modifications, no data extraction beyond what is necessary for complexity mapping.

06

Portability

The solution is designed to run on Sephora's infrastructure.

The current deployment runs on BayOne's Azure subscription for demonstration purposes. For a production engagement, the same application deploys to Sephora's Azure environment with no architectural changes. The components are:

Component Current (Demo) Production (Sephora)
Model inference BayOne Azure AI Foundry Sephora Azure AI Foundry (or existing enterprise Claude subscription)
Application hosting BayOne Azure Container Apps Sephora Azure Container Apps (or equivalent)
Data storage BayOne Azure Blob Storage Sephora Azure Blob Storage (or direct MCP connectivity to on-prem systems)
Checkpointing PostgreSQL sidecar Azure Database for PostgreSQL (managed) or sidecar
Observability LangSmith (BayOne project) LangSmith (Sephora project) or self-hosted alternative
Source connectivity Exported files via Blob Storage MCP connectors to Cognos, DataStage, SQL Server (with environment access)
No Additional Tooling Required

The solution does not introduce any proprietary frameworks, commercial middleware, or additional SaaS subscriptions beyond what Sephora already uses. LangGraph and sqlglot are open source. The application is a standard Python FastAPI service containerized with Docker. Everything runs within Azure services that Sephora's infrastructure team is already familiar with.
