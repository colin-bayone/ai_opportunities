Confidential Information Detection

Proof-of-Concept Engagement Pricing
Project Objective

Demonstrate an improved detection methodology for customer-confidential information within Escalation Solver free-text fields. This engagement assesses the current data landscape, establishes the maximum achievable detection accuracy through formal exploratory data analysis, and delivers a working detection approach benchmarked against the prior effort baseline as documented by Lam Research.

Engagement Scope
Area Detail
Application Escalation Solver (five free-text fields, 4,000 to 5,000 characters each)
Entity Types Customer name and fab identifier
Methodology Layered detection approach: deterministic pattern matching, machine learning and NLP, and generative AI for contextual edge cases, applied sequentially
Benchmarking Detection performance measured against prior effort baseline using a defined, repeatable eval protocol

Pricing is based on the scope and assumptions described in this document. Any material changes to scope, deliverable count, timeline, or technical requirements may be subject to a formal change request process with adjusted pricing.

Assumptions:

  • Lam Research provides access to Escalation Solver data, reference lists (customer names, fab/location identifiers, exclusion lists), and previously labeled examples
  • Lam Research designates one or more subject matter experts available during the POC period to define detection targets, validate results, and confirm what constitutes a true positive
  • Detection scope is five free-text fields and two entity types (customer name, fab identifier)
  • Baseline accuracy of reference data assessed during discovery; maximum achievable accuracy bounded by data quality
  • Data volume and scope agreed during discovery; POC designed for a representative data set
  • Execution environment (Lam Research infrastructure or BayOne infrastructure) agreed upon before the engagement begins, as described in the Execution Options section of this document
  • Lam Research provides timely feedback and validation during the POC period. Once the engagement is underway, if cumulative project delays beyond BayOne's control exceed 50% of the remaining engagement duration, both parties will formally review engagement terms through a change request process.
  • All data handled per confidentiality requirements; customer names redacted from BayOne documents
  • Lam Research provides documentation of prior detection effort results and target performance aims to serve as a benchmark

Exclusions:

  • Additional entity types beyond customer name and fab identifier
  • Additional applications beyond Escalation Solver
  • Production integration into Escalation Solver or any other application
  • Document and attachment scanning (POC scoped to free-text fields within escalation tickets)
  • Ground truth remediation at scale (BayOne documents findings and recommends a remediation path)
Material changes to assumptions or requests beyond stated exclusions may require a change request with adjusted pricing.

Option A (Recommended): All work is performed within the Lam Research environment. This approach ensures data never leaves Lam systems and aligns with existing confidentiality practices. BayOne will require environment access (such as a virtual machine) and access to Escalation Solver data and reference materials before the three-week timeline begins. The specific access requirements will be determined during the discovery starting point.

Option B: BayOne performs the work on its own infrastructure using data provided by Lam Research under the existing confidentiality agreement. This approach may reduce onboarding time if environment provisioning is a constraint.

The three-week POC timeline begins from the date all required access and data are in place, regardless of which option is selected.
Pricing Basis

This engagement is priced at $15,000 based on the current understanding of the Escalation Solver data, the two target entity types, and the five free-text fields as described during discovery meetings. The pricing reflects a fixed-price, outcome-based model. Pricing is the same for both execution options.
