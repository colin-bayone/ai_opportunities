Platform at a Glance

TalentAI is BayOne's internal recruitment platform, built with Django and powered by Azure cloud services. It manages the full candidate lifecycle from sourcing through hiring manager submission.

04

Foundation

These three apps form the base that every other app depends on: shared templates and utilities, authentication and authorization, and compliance audit logging.

AppPurposeKey Capabilities
core/shared Shared foundation for the entire platform Base templates (main_base, auth_base, splash_base), global template tags, CDN asset management with SRI, global search autocomplete, service registry singleton, health checks, PostgreSQL extension management (pg_trgm, pgvector), Azure email backend
core/accounts Authentication, authorization, and user management Microsoft SSO for staff, email/password for external users, magic links for candidates, 8-level role hierarchy (External through Principal), user approval workflow, staff session timeout, Microsoft Graph profile sync, 5 security middleware classes, delegated permissions
compliance/audit SOC2-compliant audit logging Write-once, immutable audit logs (cannot be modified or deleted). AuditedModel abstract base class inherited across the platform. AuditContextMiddleware captures request context per-thread. Logs: authentication events, data changes, permission modifications, management commands, access denials. Separate JobDivaAuditLog for API-specific auditing. @audit_command decorator for management commands. AuditService and utility helpers (log_access_denied, require_permission_with_audit).
05

Infrastructure

Background processing, file management, and monitoring. These apps provide the plumbing that the business logic apps rely on.

AppPurposeKey Capabilities
core/celery Background task system Four base task classes (BaseTask with auto-retry, CriticalTask with no retries for payment-safe ops, IdempotentTask safe for multiple retries, QuickTask with short timeouts). Task registry that whitelists allowed tasks with per-task config. TaskService abstraction for dispatching without direct Celery coupling. Celery Beat scheduler for periodic tasks. Redis broker. Used by 9+ other apps.
core/storage File storage management Azure Blob Storage integration, media proxy for VNet-isolated blobs, file upload/download, configurable containers per blob type
core/conversion Document conversion PDF to text extraction, format conversion pipeline, used by resume and job parsers
core/validation File type validation Magika AI-based file type detection, validates uploaded files before processing
core/diagnostics Monitoring and logging Pulsecheck request tracking, structured logging (get_logger), performance metrics middleware, WebSocket consumer for live diagnostics
06

Intelligence Layer

The AI/ML brain of the platform. These apps power semantic search, document understanding, and AI-assisted features across the recruitment workflow.

AppPurposeKey Capabilities
intelligence/ai_search Search engine powering the candidate search page Multi-strategy search: PostgreSQL full-text search, trigram similarity, fuzzy matching, pgvector semantic search, and LLM-powered features. Includes AI-assisted boolean query generation, keyword expansion, multi-vector embedding service, distance calculator for job fit scoring, embedding visualization (UMAP), and a page action registry pattern. 22 service files, 7 models, 9 management commands. Largest intelligence app at 76K+ lines.
intelligence/ai_models AI model management and orchestration Multi-provider AI model configuration, model versioning, parameter management, unified processor interface used by other apps to call AI services
intelligence/extraction Document extraction pipeline REST API (ModelViewSet at /api/v1/extraction/), document data extraction services, pipeline processing for uploaded documents
intelligence/ai_toolkit AI features hub Landing page and coordination point for AI features, discovery interface for available AI capabilities

ai_search is critical path. It directly powers the main candidate search page that recruiters use daily. The candidates app imports heavily from it: PageActionMixin, PostgresSearchUtil, AIAssistService, DistanceCalculator, MultiVectorEmbeddingService, and EmbeddingVisualizationService.

07

Recruitment Pipeline

The core business logic. The candidates app is the largest in the project at 174K+ lines and serves as the central hub that most other apps interact with.

AppPurposeKey Capabilities
recruitment/candidates Candidate management and search Single source of truth for candidate identity and profile data. 11 database models including Candidate, CandidateDocument, ComponentEmbedding, SearchConfiguration. ~27 view classes and ~35 URL patterns. Full search pipeline (keyword, Boolean, semantic, fuzzy) with caching, hydration, and paginated results. Embedding explorer (3D scatter plot), job fit visualization (radar + 3D molecular match), decomposed match API. Celery tasks for cleanup and hydration. 12 admin classes with custom sync actions. Referenced by nearly every other recruitment app. ~174K lines.
recruitment/certifications Certification ontology Certification classification and tracking, ontology mapping for candidate certifications, Celery tasks for async processing
08

Integrations

How TalentAI connects to external systems. JobDiva provides candidate data through an ETL pipeline, and Microsoft Graph handles authentication and communication.

AppPurposeKey Capabilities
integrations/jobdiva JobDiva ATS integration Full ETL pipeline syncing candidates, companies, jobs, contacts, divisions, and roles from JobDiva into TalentAI. Also supports reverse sync (TalentAI to JobDiva) for job postings. 26+ database models, dedicated API client with rate limiting, mock mode for local/stage development, admin panel with ETL dashboard for configuration and monitoring. The largest app by line count (229K+ lines). ETL internals will be covered in a dedicated separate session.
integrations/microsoft/graph Microsoft Graph API OAuth token management with automatic refresh via Celery Beat (every 5 minutes). Service layer: GraphAuthService, GraphClient, GraphUserService, GraphMailService, ProfileSyncService. Supports both delegated (user) and app-only (service account) authentication. 12 URL endpoints across user, mail, organization, and debug sections. No models of its own — uses allauth SocialToken. Consumed heavily by communications app for email sync.

JobDiva ETL is complex enough for its own session. For this overview, the key takeaway is that JobDiva is the primary source of candidate data entering the platform. The admin panel provides configuration and monitoring capabilities that the team should be familiar with.

09

Communications

How the platform communicates with users and tracks all interactions. Includes email tracking, real-time notifications via WebSocket, and a built-in bug reporting system.

AppPurposeKey Capabilities
communications Email tracking, meetings, calendar 13 database models covering email, calendar, booking, transcripts, and sync. Full email client (inbox, sent, compose, drafts, thread detail) synced via Microsoft Graph. Magic link meeting booking for candidates (unauthenticated). Meeting transcript processing pipeline. ~20 Celery tasks for email sync, calendar, and transcripts. WebSocket consumer for real-time email notifications. MS Graph webhook endpoint. 15+ service classes. ~127K lines.
user_notifications Real-time notification system Notification type registry pattern — other apps register their types (bug reports, approvals, status changes, etc.) and the system handles delivery. WebSocket consumer for real-time push. Per-user preferences with per-channel granularity. Resolution keys for grouping related notifications. NotificationMixin for models, NotificationService API for creating and pushing. HTMX-based notification panel UI.
core/bugs Bug reporting system Modal-based bug submission from any page, automatic DOM screenshot capture, native screenshot support, browser metadata collection, draft persistence with minimize/restore, severity classification, staff triage interface with quick-view panel, notification integration
