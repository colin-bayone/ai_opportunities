Icon and Image Shapes

The @{ icon: } syntax creates a node where an icon is the primary visual element, with the label displayed below. The @{ img: } syntax does the same with an external image.

graph LR ICN@{ icon: "fa:fa-server", label: "Icon Node" } ICN2@{ icon: "fa:fa-database", label: "Database Icon" } ICN3@{ icon: "fa:fa-brain", label: "AI Icon" } ICN ~~~ ICN2 ~~~ ICN3
Why the question marks? The @{ icon: } syntax uses mermaid's internal icon registry, which requires explicit icon pack registration via JavaScript (mermaid.registerIconPacks()). This is separate from the FontAwesome CSS loaded on the page. Without registration, mermaid renders a ? placeholder — the node shape itself still works correctly.

Recommended approach: For the Singularity skill, use the fa:fa-icon-name inline syntax inside standard node labels (demonstrated on the Icons Reference page). It works with just the FA CSS link and htmlLabels: true — no registration needed.

Image nodes use @{ img: "url", label: "Text", w: 60, h: 60 } and require a valid image URL. They render the image at the specified width/height with the label below. Because image nodes require an external URL, no live example is shown here — the syntax is in the reference table.
