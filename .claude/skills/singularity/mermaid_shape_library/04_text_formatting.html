All examples on this page use htmlLabels: true, which is required for FontAwesome icon rendering and HTML formatting in mermaid v11. This is the recommended mode for the Singularity skill.

Bold Text with HTML Tags

Use <b>...</b> tags for bold text within node labels when htmlLabels: true is enabled.

graph LR A["Bold Title"] B["Regular text only"] C["Bold and regular mixed"] A ~~~ B ~~~ C

Multi-line Text with Line Breaks

Use <br/> for line breaks in HTML mode. Each break creates a new line within the same node.

graph LR A["Server Name
Line 2
Line 3"] B["Database
Host: db-prod-01
Port: 5432
Version: 15.2"] A ~~~ B

Bullet Characters for Detail Lines

Unicode bullet characters create structured, list-like content inside nodes. Each character has a different visual weight and style.

graph TB B1["Small Triangle ▸
▸ Clean, modern look
▸ Recommended default"] B2["Round Bullet •
• Traditional bullet
• Familiar to all readers"] B3["Open Circle ◦
◦ Lighter weight
◦ Good for secondary items"] B4["Arrow →
→ Implies sequence

‣ Alternative to ▸
‣ Slightly different weight"] B6["En Dash –
– Subtle separator
– Minimal visual weight"] B1 ~~~ B2 ~~~ B3 B4 ~~~ B5 ~~~ B6

FontAwesome Icons in Labels

Use fa:fa-icon-name syntax to embed FontAwesome icons directly in node labels. Requires htmlLabels: true and the FA CSS loaded in the page.

graph LR A["fa:fa-server Server"] B["fa:fa-database Database"] C["fa:fa-cloud Cloud Service"] D["fa:fa-lock Secure Endpoint"] E["fa:fa-brain AI Model"] A --> B --> C A --> D --> E

Syntax Comparison: htmlLabels true vs false

Mermaid v11 supports two formatting modes. htmlLabels: true (recommended) gives you icons + HTML formatting. htmlLabels: false gives markdown-style bold but no icon rendering.

graph LR subgraph HTML["htmlLabels: true (Recommended)"] direction TB H1["fa:fa-server Bold + Icon
▸ Detail line"] end subgraph MD["htmlLabels: false (Markdown)"] direction TB M1["Icons render as text
No glyph support"] end style HTML fill:#dcfce7,stroke:#16a34a,stroke-width:2px,color:#14532d style MD fill:#fef2f2,stroke:#f87171,stroke-width:2px,color:#991b1b

Trade-off: With htmlLabels: true, use <b> for bold and <br/> for line breaks (not ** or literal newlines). With htmlLabels: false, use **bold** markdown syntax and backtick-wrapped text with literal newlines, but fa: icons will render as literal text.
