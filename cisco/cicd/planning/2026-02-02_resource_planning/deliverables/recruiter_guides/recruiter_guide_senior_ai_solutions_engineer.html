Must-Have Skills
  • 5+ years software engineering, 3+ years AI/ML, 1+ year LLMs
    Production experience, not research or coursework only
  • Python backend development
    FastAPI or Django, async patterns, APIs and data pipelines
  • LangChain or LangGraph
    Built AI pipelines with orchestration; understands chains, agents, memory
  • Prompt engineering
    Context window management, few-shot learning, structured outputs, handling failures
  • RAG and vector search
    Built retrieval systems with pgvector, Pinecone, or Qdrant; understands embeddings and chunking
  • Conversational AI
    Built chat interfaces or NL query systems; conversation history, multi-turn context
  • Testing non-deterministic systems
    Strategies for testing AI outputs that vary; evaluation approaches
  • Strong communicator
    Clear writing, comfortable presenting, explains technical concepts to non-technical audiences
  • AI pair programming proficiency
    Actively uses Claude Code, Cursor, or similar agentic AI tools daily. Not just autocomplete (Copilot) - should be comfortable with AI that writes and edits code autonomously
Nice-to-Have Background
  • Log analysis / failure diagnosis
    Experience digging through system logs to figure out why something broke
  • CI/CD tooling
    Jenkins, Airflow, GitHub Actions, GitLab CI
  • Grafana / dashboards
    Building monitoring and visualization dashboards
  • Agentic AI
    AI systems that take actions on their own, make decisions, work through problems step-by-step
  • On-prem deployment
    Deploying systems without cloud service dependencies
  • NLP fundamentals
    Older text/language processing skills, not just using ChatGPT-style APIs
  • Mentoring experience
    Leading or mentoring other engineers
Strong Signals
  • Shipped AI features that real users depend on (not just demos)
  • Can explain technical concepts clearly to non-technical people
  • Experience presenting to or working with clients/stakeholders
  • Built chat interfaces or conversational AI systems
  • Enthusiastic about AI tools - can describe specific ways Claude Code or Cursor changed their workflow
  • Has set up vector databases themselves (pgvector, Pinecone, Qdrant)
  • Mentions LangSmith, Langfuse, or similar observability tools
Warning Signs
  • Only notebook/research experience; never deployed for real users
  • Can't explain how their AI tools work, only how to configure them
  • Relies entirely on managed services (e.g., uses Azure AI Search but couldn't set up vector search without it)
  • Doesn't use AI coding tools, or only uses basic autocomplete (Copilot alone)
  • No experience working with clients or people outside their team
  • Uncomfortable presenting or gets defensive when asked to explain
  • Vague about what they personally built vs. what their team built
Screening Questions

Record all screening calls. These questions reveal depth without requiring technical evaluation. Note answers verbatim for technical review.

"When selecting a vector database for a project, what factors do you consider?"
Weak Answer
Lists database names (Pinecone, Weaviate, pgvector) without explaining WHY they'd pick one. Short answer, no reasoning.
Good Answer
Says "it depends" and explains WHAT it depends on: size of data, speed requirements, what the team already uses, budget. Weighs options.
"Tell me about a time something went wrong with an AI system in production. What happened?"
Weak Answer
Can't recall a specific situation. Gives theoretical answers ("sometimes models give wrong outputs") instead of a real story.
Good Answer
Tells a specific story with details: what broke, how they found out, what they did. You can tell they actually lived through it.
"Tell me about a time a client or stakeholder had unrealistic expectations about AI. How did you handle it?"
Weak Answer
Can't recall a specific situation. Or just agreed with whatever the client wanted. Or avoided the conversation entirely.
Good Answer
Describes a real situation: what the client expected, how they understood what was actually needed, how they communicated what was realistic.
"How do you know if an AI feature is working correctly after you ship it?"
Weak Answer
Vague answers like "we test it" or "users tell us if it's broken." No systematic process described.
Good Answer
Describes a system: how they track quality, what they measure, how they catch problems before users complain. Structured answer.
"What AI coding tools do you use, and how have they changed the way you work?"
Weak Answer
Doesn't use AI tools, or only mentions basic autocomplete like Copilot. Can't describe specific ways it changed their workflow. Dismissive about AI assistance.
Good Answer
Uses Claude Code, Cursor, or similar agentic tools. Gives specific examples of how it changed their work - faster debugging, writing tests, exploring unfamiliar codebases. Enthusiastic about the productivity gains.
"Can you explain [most complex thing on their resume] to me like I'm not technical?"
Weak Answer
Uses jargon, gets frustrated, talks down to you, or can't simplify it at all.
Good Answer
Uses analogies, checks if you're following, adjusts based on your reactions. Patient and clear. (You can evaluate this directly.)
