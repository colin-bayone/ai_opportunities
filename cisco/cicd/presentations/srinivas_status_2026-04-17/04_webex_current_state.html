Capabilities Built and Internal Tooling Landscape

BayOne has two working capabilities running locally. Cisco has two internal tools in the same space that are not yet deployed. Repository access to the internal tools has been outstanding since April 9.

BayOne-Built

Wall-E Bot and Meeting Extractor

Wall-E Running, BayOne Space
WebEx chat scraper built using the WebEx SDK. Captures structured message history (timestamp, author, email, bot flag, text, files, thread parent) with per-room scoping and deduplication.
Deployed into a BayOne-controlled WebEx space for validation. Storage is PostgreSQL hosted in a Podman container on the local development machine. No direct database access from other systems, persistence scoped to the dev machine.
Wall-E was the instrument that produced the pain point evidence on the prior three slides.
Meeting Recording Extractor Built Locally
Built against the WebEx API. Retrieves transcripts, summaries, action items, and audio files for WebEx meetings.
API constraint discovered: recording and transcript access via the API requires the meeting owner's credentials. Limits broad automation across organizationally-owned meetings.
Cisco Internal

Pulse and Scribbler

Pulse Not Deployed in NX-OS
WebEx chat scraper intended to extract messages from WebEx spaces into a database. Not deployed in the NX-OS environment. Service app deployment status unknown.
Repository access requested since April 9 and remains outstanding. Naming confusion in the GitHub teams index has further slowed identification.
Scribbler Local Only
Audio transcription tool using Whisper and Pyannote. Runs locally with no database integration and no production deployment.
Referred to as "Scribble," "Scribbler," and "Scrubber" in different internal contexts. Scribbler is the final internal name, Scrubber is a GitHub team naming artifact.

Wall-E already produced the pain point evidence. Pulse and Scribbler cover similar ground but remain inaccessible and undeployed. The direction decisions about how these tools relate come up on slide 05.
