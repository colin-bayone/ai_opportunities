WebEx Track: Proposed Architecture
WebEx Track

Proposed Unified Intelligence Architecture

Five modular layers so new data sources can be added without touching downstream components, and the application layer can be swapped per use case.

Data Sources
WebEx Chats and Spaces
Messages, threads, files, rooms, direct spaces
WebEx Meetings
Transcripts via internal tool or built-in speech to text
Future Connectors
Email, GitHub, Jira, and others
Ingestion
Service App
OAuth scoped space access, poll at interval
Pre-processor
Normalize, deduplicate, clean to model-ready format
Raw Snapshot
Original source preserved for replay and reprocessing
Storage
Unified Database
Raw and processed snapshots, indexed for semantic search, timestamped history
Multi-Source
Serves all downstream consumers from a single store
Access Layer
MCP Server
Search and query tools, context-aware retrieval
Per-Session Auth
User token scoped, never exposed to the model
Application
Agentic WebEx App
Action items, meeting summaries, compliance, chat Q and A
Swappable
Per use case, sharing the same database and MCP server
Cross-cutting Auth: OAuth 2.0 at the access boundary, user-level access control, data isolation per user, no blanket bot access. Supports both WebEx interface and direct agent access.

If the existing internal chat scraper is already running as a deployed service, the most efficient integration path is building the MCP layer directly on top of its database output rather than rebuilding ingestion from scratch.
