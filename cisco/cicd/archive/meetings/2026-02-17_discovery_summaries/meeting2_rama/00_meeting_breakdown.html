03

Problem 1: Regression Analysis

The primary pain point. 10-12 engineers spend 3-4+ hours daily analyzing test results.

"Running is okay. Running, yes, we can build so many equipment and repeat the test and everything. Analysis is the key thing. That is where the time is spent."

— Rama

The Challenge

UI timeout errors cascade across 60 test suites, affecting 2000+ test cases. Engineers manually trace back to root cause. AI could identify: "This one bug is impacting these many test cases" instantly.

"Regression analysis is one of the costly functions. Where our op-ex is being a little bit overspent, we want to manage it better or use that bandwidth to develop automation or do something else."

— Rama
04

Problem 2: UI Automation

Leadership pushing to "move the needle" on UI testing. Currently heavy on API-based testing.

Current stack uses Selenium tools with Cisco's Python automation infrastructure layer. UI changes require massive script updates.

Example Impact: One simple UI change required modifying 4,000 scripts.

"UAE [UI Automation] is not impossible, but it is a more laborious work."

— Rama
05

Problem 3: Theme Changes

Cisco moving to dark theme / night mode. Previously white background, now black.

Traffic thresholds shown as green/yellow/red need to be re-validated for the new theme. Some places modified, some not - need to catch inconsistencies across all screenshots and visual comparisons.

06

Colin's Proposed Solution

Graph Topology Approach

Build a multi-dimensional graph topology of the codebase that maps relationships between files, libraries, and dependencies. The graph is state-aware and updates as code changes.

"The trick to it is to not do it ad hoc. So for us, what we do is we'll preserve that, and it's state-aware. So whenever the code changes, the graph changes."

— Colin

Use Cases

Use Case How Graph Helps
Impact Analysis See what files a library touches, understand change impact
Test Activation Because this changed, here are the relevant tests to run
Failure Hierarchy Identify primary failure vs. cascade failures
UI Change Impact Automatically determine which scripts need updates

Crossover Opportunity: "This is actually good, because this is part of what we're doing already for Arun's team." — Colin
