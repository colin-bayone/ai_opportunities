# Categories for Analysis

## Technical Issues
1. Bug/Error
	Scope: compile/runtime errors, crashes, exceptions, regressions, defect discussions.
2. Performance Issue
	Scope: latency, throughput, timeout, high CPU/memory, bottlenecks.
3. Security Concern
	Scope: CVEs, vulnerabilities, permissions/access, credential/cert concerns.
4. Infrastructure/Deployment Problem
	Scope: CI/CD and workflow breakages like PR/xflow/codenet stuck, pipeline not triggering,
	sanity not starting, build system/environment issues.

## Feature & Development
5. Feature Request
	Scope: asks for new capabilities or explicit support requests.
6. Enhancement/Optimization
	Scope: improvements, refactors, optimization proposals.
7. Code Review
	Scope: PR links/review asks/comments/LGTM/nits.
8. Technical Debt
	Scope: legacy cleanup, workaround removal, TODO/FIXME/deprecation work.

## Project Management
9. Task Assignment
	Scope: ownership assignment, action items, requests to coordinate/ping others.
10. Timeline/Deadline Update
	Scope: ETA, due date, scheduling and milestone movement.
11. Blocker/Dependency
	Scope: blocked/waiting due to prerequisites or dependencies.
12. Status Update
	Scope: progress updates like triggered/restarted/started/resolved/passed/went fine.

## Documentation & Knowledge
13. Documentation Request
	Scope: asks for docs/wiki/readme/log locations/payload details.
14. Best Practice/Guideline
	Scope: recommended standards, conventions, and preferred approaches.
15. Knowledge Sharing
	Scope: sharing references/links/resources or informative updates.
16. Tutorial/How-to
	Scope: step-by-step guidance and how-to questions.

## Communication & Process
17. Meeting Notification
	Scope: meetings, sync calls, invites, agenda updates.
18. Policy/Process Change
	Scope: new process announcements and procedural changes.
19. Team Announcement
	Scope: broad announcements for team/group.
20. Question/Help Request
	Scope: asks for help/check/share/confirmation, "same issue for me" type requests.

## Quality & Testing
21. Test Failure
	Scope: failures/aborts in sanity/regression/testsuites and failure analysis.
22. QA/Testing Issue
	Scope: flaky tests, test environment setup issues, sanity run anomalies.
23. Release/Deployment Status
	Scope: release/build/image rollout and deployment readiness status.

## Other
24. Discussion/Brainstorm
	Scope: open-ended design/idea discussions.
25. Off-topic/General Chat
	Scope: short acknowledgements and light chat (got it, thx, check dm, sure),
	including non-text/system reactions (+1, attachment-only/non-text message).
