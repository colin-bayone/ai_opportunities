## What Set 07 Adds That Was Not Previously Recorded

Her first-team-contact date needs to move from Apr 17 to Apr 13 in the parent org chart.

### New: Two-track team structure was formalized on Apr 13, not later

Sets 02 through 06g reference the two-track structure (WebEx and build logs) as if it were an assumed operating model. Set 07 is the meeting where that structure was explicitly defined and communicated. Colin announced the split during this meeting and told each track they would have separate architecture meetings with him. This is the origin of the structure, not a mid-engagement reorganization.

### New: Wednesday escalation cadence

Prior sets had no explicit escalation schedule. Set 07 establishes Wednesday as the latest day of the week to flag access blockers to Srinivas. The rationale is Srinivas's explicit Apr 10 ask to hear about blockers earlier in the week so he can help.

### New: Weekly presentation cadence as team-owned work

Sets 05_prep and 05a document the Apr 17 presentation preparation with Colin clearly owning the slide work. Set 07 captures the plan to move presentation generation from Colin-owned to team-owned. Each track will generate architecture diagrams and Singularity-produced presentations weekly. Training was planned for the Apr 13 afternoon and subsequent days.

### New: Cisco IT flag on Saurav's Wall-E bot

Set 07 documents an email Saurav received from Cisco IT flagging his deployed Wall-E bot as non-compliant: the bot must be registered with the organization, and bots are restricted to groups between 2 and 100 users. Cisco's own NXOS WebEx group is approximately 300 users, which the bot's own policy violates. This compliance blocker does not appear in prior sets. It is not clear whether the team ever returned to this item; the subsequent Namita incident on Apr 20 reshaped the compliance landscape in a different direction (data handling violations, not bot registration).

### New: Scribbler's actual architectural state

Sets 02d (Srikar's WebEx recording extraction), 04f (Saurav's WebEx architecture), and 05_prep_pulse_scribble_naming_and_naga_stance all discuss Scribbler as if its internal architecture were partially known. Set 07 captures the specific moment Srikar clarified that Scribbler is a ~50-line Python script running Whisper locally on a selected audio file. It is not integrated with any Cisco system. Colin shifted his entire framing based on this clarification: from "they have a working tool we need to work around" to "they have a POC that BayOne's proposed architecture can replace without political friction."

### New: A/B testing three-tier methodology

Set 07 is the origin of the Scribbler-versus-WebEx A/B testing framework that later sets reference. The three tiers are (1) raw quality, (2) glossary-augmented quality, and (3) compute and scale cost. Set 07 is where this framework was articulated by Colin.

### New: Build log data model principles

Set 07 captures the star-schema call, the SHA-hash-as-key principle, the MCP-over-Git query approach, and the "persist only what cannot be regenerated" rule. Subsequent sets (02c, 04e) develop these further but Set 07 is the origin.
