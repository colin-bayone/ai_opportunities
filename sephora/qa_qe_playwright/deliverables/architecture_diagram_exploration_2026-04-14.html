Architecture Diagram Exploration

Five diagram types representing the same five-layer platform architecture

1. Mind Map

Keyword: mindmap. Radiates outward from a central concept. Good for showing relationships without implying sequence.

mindmap root((Agentic QE Platform)) Layer 1: Deterministic Automation Playwright Browser Automation Codebase Mapping Isolated Execution Docker / Azure Container Apps Layer 2: Saved Playbooks Deterministic Replay CI/CD Triggers via GitHub Actions Scheduled Execution via Airflow Release-Mapped Test Assets Layer 3: Agentic Exploration Exploratory Testing Visual Understanding Workflow Discovery Non-Destructive Guardrails Layer 4: Review and Confidence Review Agent Filtering Human-Validated Confidence Scoring Bidirectional Feedback Loop Layer 5: Observability Full Action Traceability Performance Metrics Edge Case Learning Configuration Baseline References Tolerance Settings Workflow Integration MCP Tool Connectors
DECENT: Clean hierarchy, but does not show the layered progression or feedback loops

2. Block Diagram (block-beta)

Keyword: block-beta. Arranges blocks in explicit columns. Designed for layered and grid-based layouts. Mermaid v11+.

block-beta columns 1 block:config["Configuration: Baselines | Tolerances | Workflows | MCP Connectors"]:1 end space block:l1["Layer 1: Deterministic Automation"]:1 pw["Playwright"] map["Codebase Mapping"] dock["Isolated Execution"] end space block:l2["Layer 2: Saved Playbooks"]:1 pb["Deterministic Playbooks"] cicd["CI/CD Triggers"] sched["Scheduled Execution"] end space block:l3["Layer 3: Agentic Exploration"]:1 exp["Exploratory Testing"] vis["Visual Understanding"] disc["Workflow Discovery"] end space block:l4["Layer 4: Review and Confidence"]:1 rev["Review Agent"] conf["Confidence Scoring"] end space block:l5["Layer 5: Observability"]:1 trace["Traceability"] metrics["Performance Metrics"] edge["Edge Case Learning"] end style config fill:#f8fafc,stroke:#94a3b8,stroke-width:2px style l1 fill:#dbeafe,stroke:#2563eb,stroke-width:2px style l2 fill:#dcfce7,stroke:#16a34a,stroke-width:2px style l3 fill:#ede9fe,stroke:#7c3aed,stroke-width:2px style l4 fill:#ffedd5,stroke:#ea580c,stroke-width:2px style l5 fill:#fef9c3,stroke:#ca8a04,stroke-width:2px
STRONG: Clean layers, consistent width, color-coded, no layout fighting. Best for print.

3. Architecture Diagram

Keyword: architecture-beta. Purpose-built for infrastructure and system diagrams. Mermaid v11+. Uses services, groups, and junctions.

architecture-beta group config(cloud)[Configuration Layer] group layer1(cloud)[Layer 1: Deterministic Automation] group layer2(cloud)[Layer 2: Saved Playbooks] group layer3(cloud)[Layer 3: Agentic Exploration] group layer4(cloud)[Layer 4: Review and Confidence] group layer5(cloud)[Layer 5: Observability] service playwright(server)[Playwright] in layer1 service mapping(database)[Codebase Map] in layer1 service docker(disk)[Containers] in layer1 service playbooks(server)[Playbooks] in layer2 service cicd(server)[CI/CD] in layer2 service scheduler(server)[Scheduler] in layer2 service explore(server)[Exploratory] in layer3 service visual(server)[Visual AI] in layer3 service discover(server)[Discovery] in layer3 service review(server)[Review Agent] in layer4 service confidence(server)[Confidence] in layer4 service trace(server)[Traceability] in layer5 service metrics(server)[Metrics] in layer5 service baselines(server)[Baselines] in config service tolerances(server)[Tolerances] in config service connectors(server)[MCP] in config playwright:R --> L:playbooks playbooks:R --> L:explore explore:R --> L:review review:R --> L:trace discover:B --> T:playbooks
DECENT: Infrastructure-native look, but layout control is limited and groups do not stack vertically well

4. Git Graph

Keyword: gitGraph. Shows branching and merging. Unconventional for architecture, but effective for showing how workflows fork and converge.

gitGraph commit id: "Configuration" tag: "Baselines, Tolerances, MCP" commit id: "L1: Deterministic" tag: "Playwright, Mapping, Docker" commit id: "L2: Playbooks" tag: "Saved Workflows, CI/CD, Airflow" branch exploration commit id: "L3: Explore" tag: "Exploratory Testing" commit id: "L3: Visual AI" tag: "Visual Understanding" commit id: "L3: Discover" tag: "Workflow Discovery" checkout main merge exploration id: "Validated Workflows" tag: "Back to Playbooks" commit id: "L4: Review" tag: "Review Agent, Confidence" commit id: "L5: Observe" tag: "Traceability, Metrics, Edge Cases"
CREATIVE: Shows the feedback loop naturally (branch and merge). Unusual for architecture but readable.
