What He Will Likely Ask

Based on his background, Xebia's product capabilities, and what he already knows from Set 04.

"You showed one pipeline. How does this work across hundreds of different pipeline types?"
The orchestrator identifies pipeline types from target examples. The knowledge base accumulates patterns from each approved run. Once the first few pipelines of a type are approved, similar ones move through with minimal human intervention. The exploration phase we are proposing is specifically designed to map the full complexity distribution so we can quantify that acceleration.
He asks this because Xebia's product handles 7 SQL platforms generically. He will be thinking about generalization.
"What happens when the agent encounters something it cannot translate?"
The deterministic gates catch failures before they propagate. If a gate fails, the producer retries with the specific error context injected into state. After max retries, the pipeline proceeds and flags the issue for human review. Nothing gets silently dropped. Every decision is auditable in LangSmith.
He asks this because Xebia's product "skips non-translatable code and marks with comments." BayOne's approach (retry with context) is arguably stronger here.
"Can you trace every transformation back to its source?"
Yes. Source grounding is built into every gate. The validators check that generated table names, column names, and source file references actually exist in the original artifacts. LangSmith provides the full execution trace for every API call and every node. Nothing is a black box.
He asks this because Xebia emphasizes "fully auditable reports preserving logic, lineage, and performance." He will want to see BayOne's equivalent.
"How does the Cognos MCP connectivity work technically?"
Cognos has full Java and .NET SDKs going back to version 10 (post-2003). The MCP server wraps the SDK to give agents programmatic access to report metadata, embedded SQL, and RBAC information. For this demo, we used exported XML because live MCP requires access to the Cognos environment. The exploration phase would include validating the MCP connector against Sephora's specific Cognos version.
He has deep Cognos expertise and 8 years at IBM. He can assess this claim technically. Be precise. Do not handwave.
"Does the generated output actually conform to the existing framework?"
The config generator copies the target templates provided by Sephora's team exactly and only changes values where the pipeline data requires it. The gate validators check that YAML token names, HQL column types, and deployment config patterns match the target examples. The output is pipeline YAML, deployment YAML, and HQL, exactly as Sergey specified. No notebooks, no new framework.
He is the Enterprise Architect for Sephora's Data Platform. He will check whether the output fits what he maintains.
"What model are you using and how do you handle token costs at scale?"
Claude Opus for heavy reasoning tasks (pattern mapping, SQL generation planning, dependent stage generation). Claude Sonnet for lighter tasks (orchestration, independent stage generation, config generation). Both via Azure AI Foundry, which is significantly cheaper than commercial API. Deterministic gates are zero LLM cost. The architecture minimizes token usage by doing as much as possible with Python and sqlglot before involving the LLM.
As an architect, he will think about production viability. Token costs at 6,000 reports is a real question.

How to Handle Him in the Demo

  • Acknowledge him warmly. He has been part of this since the technical deep dive. "Good to see you again, Maher."

  • Welcome his questions. Treat them as validation opportunities, not challenges. His approval carries weight with Andrew and Grishi.

Do Not

  • Do not mention Xebia or their product. Do not acknowledge the competitive overlap. Let him draw his own conclusions.
  • Do not position this as a "product." That is Xebia's lane. BayOne builds custom solutions. The word "product" hands them the comparison.

  • Do not be defensive if he asks hard questions. He is doing his job. A defensive response signals insecurity. A precise response signals competence.
  • Do not speculate about what Xebia can or cannot do. Stay in BayOne's lane. Talk about what was built, not what others have not built.
