What the Client Cared About

Based on where they spent their questions and what generated the most engagement.

Their Top Priorities (by question volume)

1. Security and Azure isolation. Maher's first major question. The moment Colin confirmed everything stays within Azure, the entire security conversation ended. This was clearly a pre-condition for them.

2. Validation methodology. Both Maher and Grishi wanted to understand how the system catches errors. The deterministic-first, LLM-second approach resonated. Maher specifically asked about the retry loop and auto-fix.

3. Scalability across business areas. Andrew wanted confirmation that this can run in parallel across all subject areas, not one report at a time. Colin's "orchestrator of orchestrators" concept landed.

4. Output compatibility with their existing framework. The pipeline produces YAML configs for the AggregationApplication framework, not notebooks or new code. This was established in prior meetings and confirmed in the demo.

5. Portability and self-sufficiency. Vishal's primary concern. Can Sephora run this without BayOne? Can they modify agents? What happens when the migration is done?

What They Did NOT Ask About

Notably absent from the conversation: cost modeling, pricing, accuracy measurement methodology, Cognos report-level testing, rollback strategy, timeline estimates, and model selection details. This suggests the demo was purely a technical validation gate, with the commercial conversation happening on a separate track with Mani.

Our Differentiators

What sets BayOne apart in this engagement, as demonstrated in the meeting.

Custom Solution, Not a Product
Everything was built specifically around Sephora's DataStage XML, stored procedures, schemas, and target framework in two weeks. A generic migration tool would produce notebooks or raw SQL that does not fit their existing support model.
Deterministic-First Validation
Every gate runs Python and sqlglot checks at zero LLM cost before the language model is involved. This is the architecture that Maher validated. Generic tools do not have this rigor.
Framework-Compatible Output
The pipeline produces YAML configs for the existing AggregationApplication framework. Not spaghetti code notebooks. Sergey (their IBM tools SME) was emphatic about this requirement, and BayOne delivered.
Knowledge Base Learning
Approved patterns are saved and improve future runs. The system gets smarter with every migration. Generic tools start fresh every time.
100% Azure, Zero External Calls
Everything runs within Azure AI Foundry, Container Apps, and Blob Storage. No data leaves their network. Deployable to their VNet as-is. IT security objection eliminated.
Human-in-the-Loop with Auto-Fix
AI-powered review with surgical code corrections. Not a black box. The confidence score and issue classification (warnings, not errors) give the team transparency and control at every step.
Colin's Prior Experience
Colin led the same type of EDW modernization at Coherent Corp (40K employees, $1.6B revenue, SSAS/SSRS to Snowflake). He has done this before. He is not learning on the job.
No Vendor Lock-In
The system is temporary for the migration. Agents are customizable. Sephora keeps everything. BayOne's goal is to make itself unnecessary for the migration tooling once the program is complete. Maher (their own architect) confirmed this framing.
