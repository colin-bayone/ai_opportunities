## What Was New in Set 06

1. **Complete pipeline architecture documented.** Saurav walked through every stage in detail: Orchestrator, G1, three parallel parsers (DataStage/SQL/DDL each with deterministic + LLM), G2, Pattern Mapper (single Opus call), three-step SQL generator (plan, independent stages parallel, dependent stages), G3/G4 (deterministic + LLM business logic at 75% threshold), three parallel config generators (pipeline YAML, deployment YAML, HQL), config gate, validation, AI review, auto-fix, approval with knowledge base learning.

2. **Auto-fix feature.** New since Set 05. Works like Claude Code: treats generated SQL and all artifacts as files, has tools to read and edit, makes surgical edits based on review findings. Alternative to full regeneration (reject-and-retry).

3. **AI Review button.** New since Set 05. Sends all validation details to AI for an audit. Finds false positives and missed issues. Output is descriptive enough for data engineers to act on.

4. **Knowledge base learning.** After human approval, patterns and decisions are saved to a JSON knowledge base. Future runs learn from these approved patterns. The system improves with use.

5. **LangSmith tracing is connected.** Shows waterfall view of all API calls and node executions. Not real-time but within minutes. Blue = stage level, orange = LLM level. Can see input/output for any node. This is the technical audience's deep-dive tool.

6. **Execution log with live spinners.** Shows what each agent is doing with a timer. Updates when agents complete stages. Much more engaging than the static dropdowns from Set 05.

7. **Known issues cataloged.** Reject-and-retry breaks DAG animation (use auto-fix instead), zip download broken (use individual downloads), pipeline_runs table missing on Colin's DB (Saurav fixing), advisory text rendering bug, duration includes human wait time.
