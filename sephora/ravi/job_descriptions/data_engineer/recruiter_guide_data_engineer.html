Recruiter Guide: Data Engineer

Remote (Offshore, India preferred) | Experience: 3-7 years | Azure only

Role Summary

Data engineering role focused on pipelines, ETL, and infrastructure that feeds ML systems. This is NOT an admin or IT role. Candidates should have hands-on experience building and operating data systems in an MLOps context, not just managing Hadoop clusters or Kafka brokers.

Hard Requirement: AI pair programming tools (Claude Code, Cursor, or Windsurf). GitHub Copilot is NOT sufficient. Must ask about this in screening. Candidates should discuss specific benefits and weaknesses of their tool, not generic answers.

What to Look For

Strong Signals

  • Data pipelines, ETL at scale
  • Spark, Kafka, Databricks experience
  • Azure data platform experience
  • SQL and NoSQL databases in production
  • Data modeling for ML use cases
  • Feature store familiarity
  • Open source tooling background
  • Experience supporting ML teams
  • Workflow orchestration (Airflow, etc.)

Weak Signals (De-prioritize)

  • Primary background as Hadoop admin
  • Primary background managing Kafka
  • Traditional DBA or IT admin roles
  • AWS-only or on-prem-only experience
  • No MLOps or ML context
  • Only ML experience, no data engineering
  • Cannot discuss pipeline architecture
  • Dismissive of AI pair programming tools

Screening Questions

  • "Walk me through a data pipeline you built end-to-end. How did data flow from source to consumption?"
  • "What AI pair programming tools do you use? Walk me through your typical workflow. What are its strengths and weaknesses?"
  • "What is your experience with Azure's data platform? If not Azure specifically, what cloud data platforms have you used?"
  • "Have you worked with ML teams before? How did your pipelines feed into model training or serving?"
  • "Tell me about your experience with Spark, Kafka, or Databricks. Which have you used most, and in what context?"

Key Points

  • Pipeline-oriented, not admin-oriented. They should build systems, not just manage infrastructure.
  • Must have MLOps context. This role supports ML systems, not standalone data warehousing.
  • Azure only. Reject AWS-only or on-prem-only candidates.
  • Open source background expected. Most tools in this space are open source.
  • Reject candidates with only ML experience and no data engineering.
  • Should be confident and able to discuss architecture decisions.
