Recruiter Guide: AI Engineer

Remote (Offshore, India preferred) | Experience: 2-5 years | Python primary

Role Summary

AI-assisted development specialist who builds automation using Claude Code skills, agents, and sub-agents. This is about AI tooling and workflow automation, not traditional ML or data engineering. The ideal candidate lives and breathes AI pair programming tools and stays current on the rapidly evolving landscape.

Core Requirement: Expert-level proficiency with AI pair programming tools is the foundation of this role. Candidates must demonstrate deep, hands-on experience with Claude Code, Cursor, or Windsurf. GitHub Copilot is NOT sufficient. They should have opinions and be able to discuss tradeoffs.

What to Look For

Strong Signals

  • Claude Code skills or agent development
  • LangChain, LangGraph, or Claude Agent SDK
  • Workflow automation with AI tools
  • Strong Python fundamentals
  • GitHub, API integrations
  • Stays current on AI tooling releases
  • Good judgment on AI vs deterministic
  • Proactive, ownership mentality
  • Fast-paced team experience

Weak Signals (De-prioritize)

  • No AI pair programming experience
  • Dismissive of AI-assisted development
  • Only traditional software engineering
  • Cannot discuss AI tool tradeoffs
  • Passive, waits for direction
  • Unfamiliar with current AI tooling
  • Poor communication skills
  • No Python experience

Screening Questions

  • "What AI pair programming tools do you use daily? Walk me through your typical workflow."
  • "What are the specific strengths and weaknesses of [tool they mention]? When does it fail you?"
  • "Have you built any Claude Code skills, agents, or automation? Tell me about them."
  • "How do you decide when to use AI-assisted approaches versus writing code yourself?"
  • "How do you ensure code generated with AI assistance is reliable and maintainable?"

Key Points

  • This is about AI tooling proficiency, not ML model development.
  • Candidates should be excited about AI-assisted development, not skeptical.
  • Look for proactive problem solvers who take ownership, not passive executors.
  • Strong communication is essential; this team has intensive collaboration sessions.
  • Python is the primary language; must have solid fundamentals.
  • Should be current on what tools exist and have opinions on tradeoffs.
