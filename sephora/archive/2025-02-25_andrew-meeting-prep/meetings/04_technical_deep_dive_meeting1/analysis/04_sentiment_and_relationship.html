Transparency About Process

Andrew provided an exceptionally detailed walkthrough of the manual workflow, sharing internal process details openly.

"Now I'm gonna throw the ball back to you. Now that you know the flow, my expectation is... is there any way throughout this step that we can create agents?"

High trust signal. Andrew is sharing the "how" so BayOne can genuinely help, not just playing vendor games.

Vision Alignment

Andrew's articulation of the agent swarm vision closely matches what BayOne has been proposing.

"I'm not expecting one agent do it... it's like a lot of smaller agents... and then you have that one orchestrator to just orchestrating this whole entire all the smaller agents."

Strong alignment on approach. Andrew has bought into the vision and is advocating for it internally.

Gariashi Chakrabarty

Director, Data Engineering, Sephora

Direct About Disappointment

Gariashi surfaced the expectation mismatch directly rather than letting it simmer. Notably, she was the only Sephora participant who expected a demo — Andrew, Maher, and Sergey were aligned with the technical discussion format.

"I think we were excited to see the demo today."

Gariashi is direct and honest. This is healthier than passive disappointment. She wants to see proof but isn't hostile about it. The fact that she was the only one out of the loop suggests an internal communication gap on Sephora's side, not a BayOne failure.

Pragmatic Acceptance

After the pivot, Gariashi engaged constructively and agreed to next steps.

"Yeah, that sounds good."

Gariashi adapted to the revised meeting purpose. Needs demo to gain confidence but isn't blocking progress.

04

Maher Burhan

Enterprise Architect, Sephora

Practical Problem-Solver

Maher Burhan's first instinct was to find workarounds, not create obstacles.

"The agent can do the work, the migration work, without having access, without validating whether it's working or not."

Maher Burhan is a pragmatist. He understands organizational constraints and works around them rather than using them as excuses.

Honest About Constraints

Maher Burhan openly shared past failures and security blocks.

"We tried to run our own agent on the independent record, we get blocked by security."

Trust signal. Sharing failures shows transparency and sets realistic expectations.

Need for Proof

"We need to see a demo, really. To see if this will work or not. Because we didn't have success."

Healthy skepticism from experience. Maher Burhan will be convinced by results, not promises.

05

Sergey Shtypuliak

SME - IBM Tools - Consultant, Sephora

Grounded Skepticism

Sergey's comment about difficulty came from deep experience with these tools.

"It's easy said than done. I worked with Collins, I wrote code for the BI server, and that's a complicated word."

Not hostile, just realistic. Sergey has seen promises before and wants to see execution. The demo will need to impress him specifically.

Constructive Engagement

Despite skepticism, Sergey provided extremely specific and useful requirements for the agent output.

"Main question right now, how to teach these agents to understand that, okay, we need to create three YAML files with some configuration inside."

Sergey wants this to work. He's giving the roadmap for success, not creating obstacles.

Vision of Ideal State

"If we can ask him... hey man, we have 10 jobs, please convert them, generate YAML files, commit them to repo. After that, some developer can quickly review that it's good and that's it. So this is like ideal world."

Sergey can envision the future state and is excited by it. He just needs to see it work.
